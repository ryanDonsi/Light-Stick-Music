# 📊 ODF 매개변수 완전 가이드

각 파라미터가 ODF 곡선과 BPM 탐지에 미치는 영향을 시각적으로 설명합니다.

---

## 🔄 ODF 계산 파이프라인

```
음성 파일
    ↓
[1단계] IIR 필터링 → 저/중/전역 밴드 추출
    ↓
[2단계] 다중 대역 ODF 계산 (각 밴드별)
    ├─ Moving Average (Smooth Window) ← 파라미터 1️⃣
    ├─ Positive Diff (상승 감지)
    └─ Local Normalize Max (Local Window) ← 파라미터 2️⃣
    ↓
[3단계] 가중 결합 (1.0 + 1.8 + 0.8)
    ↓
[4단계] 최종 정규화 (Global Window) ← 파라미터 3️⃣
    ↓
최종 ODF 곡선
    ↓
[5단계] BPM 탐지
    ├─ Autocorrelation 계산
    ├─ Log-normal Prior (Center, STD) ← 파라미터 4️⃣, 5️⃣
    └─ Best Lag 선택 (Min/Max Beat) ← 파라미터 6️⃣, 7️⃣
    ↓
검출 BPM
```

## 📋 각 파라미터 상세 설명

### 1️⃣ **Smooth Window** (1-10, 기본값: 3)

#### 역할
🔊 **신호 평활화** - ODF의 노이즈를 제거하고 부드럽게 만들기

#### 작동 원리
```
Smooth Window = 3일 때:
output[i] = 평균(input[i-1], input[i], input[i+1])

Smooth Window = 5일 때:
output[i] = 평균(input[i-2], input[i-1], input[i], input[i+1], input[i+2])
```

#### 👀 시각적 효과

```
값 작음 (Smooth Window = 1):
ODF 곡선
█│  █  █  █ ▄█│  (불규칙한 spike들)
│ █│ █ █ █││ █
───────────────

값 중간 (Smooth Window = 3) - 권장:
ODF 곡선
  │    ▂▄▆█▆▄▂   (부드러운 peak)
  │   ▄█       █▄
───────────────

값 큼 (Smooth Window = 7):
ODF 곡선
     │   ▆█▆      (과도하게 평탄화됨)
     │  █   █
──────────────
```

#### 💡 언제 조정할까?

**증가시키면** (3 → 5-7):
- ✅ 노이즈가 많은 노래
- ✅ 저음역대 신호가 약한 곡
- ❌ 비트가 명확한 빠른 곡에서는 정확도 하락

**감소시키면** (3 → 1-2):
- ✅ 비트가 명확한 곡
- ✅ 빠른 음악 (≥140 BPM)
- ❌ 노이즈가 많으면 오류 증가

**유지** (3):
- ✅ 대부분의 곡에 최적
- ✅ 균형잡힌 설정

---

### 2️⃣ **Local Window** (30-120, 기본값: 60, 단위: 프레임)

#### 역할
🎯 **로컬 정규화 윈도우** - 각 시점의 신호를 주변과 비교하여 정규화

#### 작동 원리
```
각 프레임 i에서:
1. 주변 Local Window 범위 내의 최댓값을 찾기
2. 해당 프레임의 값을 최댓값으로 나누기
3. 결과 = 0~1 사이의 정규화된 값

Local Window = 60 프레임 = 3초 (50ms hop 기준)
```

#### 👀 시각적 효과

```
작은 값 (Local Window = 30):
ODF 곡선
█│                 (큰 값만 생존)
│ █│█ █│█ █│█
────────────────
(많은 부분이 0에 가까워짐 - 신호 손실)

중간 값 (Local Window = 60) - 권장:
ODF 곡선
  ▂▄█▄▂  ▂▄█▄▂    (온셋마다 peak 보존)
█▀     ▀█▀     ▀█
────────────────

큰 값 (Local Window = 100):
ODF 곡선
▅▆█▆▅  ▅▆█▆▅    (온셋 구분 어려움)
▅▆█▆▅  ▅▆█▆▅
───────────────
(모든 부분이 높게 정규화됨 - 선택성 감소)
```

#### 💡 언제 조정할까?

**감소시키면** (60 → 30-40):
- ✅ 각 비트가 명확한 곡
- ✅ 빠른 곡 (≥140 BPM)
- ❌ 느린 곡에서 신호 손실

**증가시키면** (60 → 80-100):
- ✅ 느린 곡 (<80 BPM)
- ✅ 음역대가 넓은 곡
- ❌ 비트 구분이 애매해짐

**유지** (60):
- ✅ 기본값이 잘 작동
- ✅ 대부분의 BPM 범위에서 좋음

---

### 3️⃣ **Global Window** (30-120, 기본값: 80, 단위: 프레임)

#### 역할
🌍 **배경 제거 + 최종 정규화** - 전체 곡의 배경 소음을 제거

#### 작동 원리
```
1단계: 배경 제거
각 프레임에서 로컬 평균을 빼기
new_value = max(0, value - local_mean)

2단계: 최종 정규화
로컬 윈도우 내 최댓값으로 나누기
```

#### 👀 시각적 효과

```
작은 값 (Global Window = 30):
ODF 곡선
█│        █│        (배경이 많이 남음)
│ █│█ █│█ │ █│█ █
─────────────────
(높은 베이스라인)

중간 값 (Global Window = 80) - 권장:
ODF 곡선
▂▄█▄▂  ▂▄█▄▂      (깨끗한 베이스라인)
▄       ▄       ▄
─────────────────

큰 값 (Global Window = 120):
ODF 곡선
█▄▂  █▄▂         (과도한 제거)
▁      ▁
─────────────────
(약한 비트까지 제거됨)
```

#### 💡 언제 조정할까?

**감소시키면** (80 → 40-60):
- ✅ 배경 음악/합창이 있는 곡
- ✅ 저음역이 강한 곡
- ❌ 약한 신호가 더 약해짐

**증가시키면** (80 → 100-120):
- ✅ 느린 곡의 배경 제거
- ✅ 음악이 단순한 곡
- ❌ 약한 온셋이 사라질 수 있음

**유지** (80):
- ✅ 기본값이 잘 작동
- ✅ 배경과 신호의 균형

---

### 4️⃣ **Prior Center (ms)** (300-800, 기본값: 500)

#### 역할
🎵 **BPM 선호 중심값** - 어느 BPM을 중심으로 선호할지 설정

#### 작동 원리
```
500ms = lag 10 (50ms hop 기준)
     = 60000 / (10 × 50) = 120 BPM

다른 값들:
300ms → 200 BPM (빠른 곡 선호)
400ms → 150 BPM
500ms → 120 BPM (기본)
600ms → 100 BPM
700ms → 86 BPM
800ms → 75 BPM (느린 곡 선호)
```

#### 📊 가우스 분포 시각화

```
Prior Center = 300ms (빠른 곡 중심):
           ▂▄█▄▂
점수   ──▂█     █▂──
     50  75 100 125 150 175 200 BPM
             ↑
           중심 (200 BPM)

Prior Center = 500ms (표준, 중간 속도):
           ▂▄█▄▂
점수   ──▂█     █▂──
     50  100 120 140 160 180 200 BPM
             ↑
           중심 (120 BPM)

Prior Center = 700ms (느린 곡 중심):
           ▂▄█▄▂
점수   ──▂█     █▂──
     50  60  86  110 130 150 BPM
             ↑
           중심 (86 BPM)
```

#### 💡 언제 조정할까?

**감소시키면** (500 → 350-400):
- ✅ 빠른 곡이 많음 (≥130 BPM)
- ✅ K-pop, EDM 스타일
- ❌ 느린 곡에서 Half-tempo 오류 증가

**증가시키면** (500 → 600-700):
- ✅ 느린 곡이 많음 (<100 BPM)
- ✅ 발라드, 클래식
- ❌ 빠른 곡에서 Double-tempo 오류 증가

**유지** (500):
- ✅ 기본값 (120 BPM 중심)
- ✅ 대부분의 곡에 최적

### 5️⃣ **Prior STD (octave)** (0.5-4.0, 기본값: 2.0)

#### 역할
📈 **BPM 분포의 폭** - Prior 분포가 얼마나 넓을지 설정

#### 작동 원리
```
옥타브 단위: log₂ 스케일
1 옥타브 = BPM 2배/반배

STD = 1.0 → 좁은 분포
STD = 2.0 → 중간 분포 (권장)
STD = 3.0 → 넓은 분포
STD = 4.0 → 매우 넓은 분포
```

#### 📊 분포 시각화

```
STD = 1.0 (좁은 분포):
점수  █
      █▄▂
      60 80 100 120 140 160 180 BPM
             ↑
           120±60 BPM만 높음
           (60~240 BPM 범위)

STD = 2.0 (권장):
점수   ▂▄█▄▂
      ───█────
      40 60 80 100 120 140 160 180 200 BPM
             ↑
           120±2octaves (30~480 BPM)

STD = 3.0 (넓은 분포):
점수 ▁▂▃█▃▂▁
    ────────
    20 40 60 100 150 200 240 BPM
         ↑
       매우 넓은 범위

STD = 4.0 (매우 넓음):
점수 ▁▂▃▄█▄▃▂▁
    ──────────
    10 30 60 100 200 300 BPM
         ↑
       거의 모든 BPM 동등
```

#### 💡 언제 조정할까?

**감소시키면** (2.0 → 0.5-1.0):
- ✅ 특정 BPM이 확실함
- ✅ Half-tempo/Double-tempo 오류 제거
- ❌ 예상과 다른 BPM은 탐지 못함

**증가시키면** (2.0 → 3.0-4.0):
- ✅ 느린 곡에서 Half-tempo 오류 감소
- ✅ 다양한 BPM 범위 커버
- ❌ 잘못된 BPM을 선택할 수 있음

**유지** (2.0):
- ✅ 최적값 (43곡 중 35곡 정확)
- ✅ 좋은 균형

### 6️⃣ **Min Beat (ms)** (250-500, 기본값: 375)

#### 역할
⚡ **최소 비트 간격** = 최대 BPM 제한

#### 작동 원리
```
Min Beat = 375ms
→ 최대 탐지 BPM = 60000 / 375 = 160 BPM

다른 값들:
250ms → 240 BPM 최대
300ms → 200 BPM 최대
375ms → 160 BPM 최대 (기본)
400ms → 150 BPM 최대
500ms → 120 BPM 최대
```

#### 💡 언제 조정할까?

**감소시키면** (375 → 250-300):
- ✅ 빠른 곡이 많음 (≥150 BPM)
- ✅ K-pop, EDM
- ❌ 느린 곡에서 오류 증가

**증가시키면** (375 → 400-500):
- ✅ 느린 곡이 많음
- ✅ 클래식, 발라드
- ❌ 빠른 곡을 놓칠 수 있음

**유지** (375):
- ✅ 기본값 (최대 160 BPM)
- ✅ 대부분의 곡 커버

### 7️⃣ **Max Beat (ms)** (800-1500, 기본값: 1000)

#### 역할
🐢 **최대 비트 간격** = 최소 BPM 제한

#### 작동 원리
```
Max Beat = 1000ms
→ 최소 탐지 BPM = 60000 / 1000 = 60 BPM

다른 값들:
800ms → 75 BPM 최소
1000ms → 60 BPM 최소 (기본)
1200ms → 50 BPM 최소
1500ms → 40 BPM 최소
```

#### 💡 언제 조정할까?

**감소시키면** (1000 → 800-900):
- ✅ 빠른 곡이 많음
- ✅ 느린 곡이 없을 때
- ❌ 느린 곡이 탐지되지 않음

**증가시키면** (1000 → 1200-1500):
- ✅ 매우 느린 곡 (≤50 BPM)
- ✅ 느린 발라드, 클래식
- ❌ 분석 시간 증가

**유지** (1000):
- ✅ 기본값 (최소 60 BPM)
- ✅ 대부분의 곡 커버

## 🔧 실전 조정 가이드

### Step 1: 기본 설정으로 시작
모든 파라미터를 기본값으로 설정 후 분석

### Step 2: ODF 곡선 확인
**[결과]** 탭에서 ODF 곡선 관찰:
- 💚 **비트마다 명확한 peak** → 정상
- 🟡 **peak가 불규칙** → Smooth Window 증가
- 🟡 **배경이 높음** → Global Window 증가
- 🔴 **peak가 거의 없음** → 신호 자체가 약함

### Step 3: AC/Prior/Score 그래프 확인
**AC 곡선** 관찰:
- 💚 **정확한 lag에서 peak** → BPM 정확
- 🟡 **2배/반배 lag에서 peak** → STD 조정 필요
- 🔴 **여러 lag에서 비슷한 높이** → 신호가 약함

### Step 4: 배치 분석으로 검증
여러 곡을 한번에 분석하여 정확도 확인

### Step 5: 세밀 조정
- 정확도 높이기 위해 한 가지씩만 조정
- 매번 배치 분석으로 전체 영향 확인

## 📈 최적화 팁

### ✅ 효과적인 조정
1. **한 번에 하나만** 조정하기
2. **배치 분석으로 검증** 필수
3. **부정확 곡의 ODF 분석** (AC 신호 강도 확인)
4. **결과 저장** (JSON으로 조합별 기록)

### ❌ 피해야 할 것
1. **모든 파라미터를 동시에** 조정 ✗
2. **한 곡만 보고** 판단 ✗
3. **극단적인 값** (Min-Max 끝값) ✗
4. **기본값을 과신** (곡에 따라 다름) ✗

## 📊 파라미터 간 관계

```
ODF 신호 강도 향상:
Smooth Window ↑ + Local/Global Window ↑ = 신호 강화

BPM 탐지 정확도:
Prior STD ↑ = 더 다양한 BPM 허용 (정확도 ↑, 하지만 오류 가능성도 ↑)

느린 곡 vs 빠른 곡:
Prior Center + Min/Max Beat = BPM 범위 조정
```

## 🎓 이해하기

### 왜 "기본값 2.0"이 최선일까?
```
PRIOR_STD_OCTAVE 분석:
- 1.0: 정확하지만 제한적 (중간속도 곡에만 좋음)
- 2.0: 균형잡힌 선택 (대부분의 곡 커버) ✅
- 3.0+: 포용적이지만 오류 가능성 증가
```

### "Half-tempo" 오류란?
```
실제 BPM: 120 BPM (lag = 10)
검출: 60 BPM (lag = 20, 즉 2배)
→ AC가 lag 20에서 더 높음
→ Prior STD를 확대하면 lag 10도 고려
```

## 📌 요점 정리

| 파라미터 | 역할 | 증가시 | 감소시 |
|---------|------|--------|--------|
| Smooth | 노이즈 제거 | 부드러움 ↑ | 민감도 ↑ |
| Local | 온셋 구분 | 선택성 ↓ | 신호 손실 ↑ |
| Global | 배경 제거 | 배경 ↓ | 신호 손실 ↑ |
| Prior Center | BPM 중심 | 느린쪽 선호 | 빠른쪽 선호 |
| Prior STD | BPM 범위 | 포용적 | 선택적 |
| Min Beat | 최대 BPM | 빠른곡 불가 | - |
| Max Beat | 최소 BPM | - | 느린곡 불가 |
