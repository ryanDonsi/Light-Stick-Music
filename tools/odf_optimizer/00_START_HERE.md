## 🎯 주요 기능

### ✅ 완성된 기능

#### 1. 파일 등록
- 파일 다이얼로그로 MP3/WAV/FLAC 등록
- Beat Accuracy Checker처럼 파일 목록 관리
- 여러 곡 동시 등록

#### 2. 실시간 매개변수 조정
```
7개 주요 매개변수를 슬라이더로 조정:
- Smooth Window (1-10)
- Local Window (30-120 프레임)
- Global Window (30-120 프레임)
- Prior Center MS (300-800)
- Prior STD (0.5-4.0 옥타브)
- Min Beat MS (250-500)
- Max Beat MS (800-1500)
```

#### 3. 분석 실행
- 백그라운드 QThread로 처리 (UI 응답성 유지)
- 첫 분석: 20초 ~ 3분
- 이후 분석: 빠름 (메모리 캐시)

#### 4. 결과 시각화
```
[결과] 탭:
- ODF 곡선 그래프
- AC/Prior/Score 관계도
- MADMOM_GT 비교 (✅/❌)
- BPM 및 오차율 표시
```

#### 5. 배치 분석
```
[배치 분석] 탭:
- 등록된 모든 곡 한번에 분석
- GT BPM과 비교
- 전체 정확도 계산
- 결과 테이블 출력
```

#### 6. 결과 내보내기
- JSON 형식으로 저장
- 매개변수 + 검출 BPM + 오차율 기록
- 타임스탬프 자동 추가

## 🔧 기술 스펙

### 구현

| 항목 | 사양 |
|------|------|
| 언어 | Python 3.8+ |
| GUI 프레임워크 | PyQt6 |
| 오디오 처리 | librosa (28kHz, mono) |
| ODF 계산 | Kotlin BeatDetectorV1 포트 |
| 시각화 | matplotlib |
| 처리 방식 | 백그라운드 QThread |

### ODF 파이프라인

```
음성 파일 (MP3/WAV/FLAC)
    ↓ librosa.load(28kHz, mono)
    ↓
IIR 필터링 (저/중/전역 밴드 분리)
    ├─ Low:  LOW_ALPHA=0.12
    ├─ Mid:  MID_LP1_ALPHA=0.35, MID_LP2_ALPHA=0.08
    └─ Full: 풀 대역
    ↓
다중 대역 ODF 계산 (각 대역)
    1. Moving Average (평활화)
    2. Positive Diff (상승만 감지)
    3. Local Normalize Max (정규화)
    ↓
가중 결합 (low×1.0 + mid×1.8 + full×0.8)
    ↓
최종 정규화 (Global Window)
    ↓
BPM 탐지
    1. 자동상관 계산
    2. 로그-정규 사전확률 적용
    3. 최고 점수 lag 선택
    4. BPM 변환 (60000 / (lag × hop_ms))
    ↓
결과 (BPM, ODF, AC/Prior/Score)
```

### 로그-정규 Prior 분포

```
중심: 500ms (≈ 120 BPM)
분포: log-normal
수식: 
  log_ratio = log₂(lag_ms / 500)
  prior = exp(-0.5 × (log_ratio / std_octave)²)
  score = ac × prior

STD 효과:
  - STD=1.0: 좁은 분포 (120 BPM 중심)
  - STD=2.0: 넓은 분포 (60-240 BPM 범위)
  - STD=3.0+: 매우 넓은 분포 (느린 곡 최적화)
```

```
1. GUI 실행
2. 파일 등록 (📁 파일 추가)
3. 곡 선택
4. 분석 실행 (🔄 분석 시작)
5. 결과 확인 ([결과] 탭)
6. 필요시 매개변수 조정
7. 반복 (3~6단계)
```

### 최적화 워크플로우
```
1. 기본 설정으로 배치 분석
   → 초기 정확도 기록 (예: 35/42 = 83.3%)

2. 문제 곡 식별
   → MADMOM_GT 비교에서 부정확 곡 찾기
   → ODF 신호 분석 (AC 강도, 형태)

3. 가설 수립
   → "AC 신호가 약함" → 평활화 증가
   → "Half-tempo 오류" → Prior STD 조정
   → "느린 곡 오류" → Prior Center 증가

4. 매개변수 조정 및 테스트
   → 한 가지만 조정
   → 배치 분석으로 결과 확인
   → 개선 여부 판단

5. 최적 설정 선택
   → 가장 높은 정확도
   → 부작용 최소 (다른 곡 악화 없음)
   → BeatDetectorV1.kt에 적용
```

## 💡 팁

### 효율적인 최적화
1. **작은 파일로 시작**: 큰 파일은 분석이 오래 걸림
2. **한 번에 한 가지만**: 여러 매개변수를 동시에 조정하면 혼란스러움
3. **배치 분석으로 검증**: 한 곡의 개선이 다른 곡을 악화시킬 수 있음
4. **결과 기록**: JSON 내보내기로 각 조합 기록

### 신호 분석
- **ODF 곡선 높이**: 신호 강도 → 높을수록 좋음
- **Peak 위치**: BPM 대응 → 정확한 lag에 peak 있어야 함
- **AC/Prior/Score**: Score가 높은 lag이 선택됨

### 문제해결
- 모든 곡이 부정확 → 매개변수를 기본값으로 리셋
- 특정 곡만 틀림 → 그 곡의 ODF 신호 분석
- GUI 응답 없음 → 분석 진행 중 (대기) 또는 파일 손상
