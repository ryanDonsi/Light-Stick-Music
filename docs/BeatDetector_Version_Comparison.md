# BeatDetector 버전별 성능 비교
## BeatDetector Version Performance Comparison

**비교 기준**: 동일한 33곡 데이터셋, 3개 워커 스레드 멀티스레드 환경  
**측정 날짜**: 2026-06-23  

---

## 📊 성능 비교 테이블

| 항목 | BeatDetectorV0 | BeatDetectorV1 | BeatDetectorV2 |
|------|----------------|----------------|----------------|
| **설명** | 간단한 비트 감지 | 중간 정확도 | 높은 정확도 (추천) |
| **입력 타입** | Envelope (사전 계산) | PCM 직접 입력 | 스트리밍 청크 |
| **hopMs** | 50ms (낮은 분해능) | 10ms (높은 분해능) | 10ms |
| **처리 알고리즘** | IIR 3밴드 ODF + AC | IIR 3밴드 ODF + AC + Log-Normal Prior | Dual ODF + DP |
| **메모리 사용** | 낮음 | 중간 | 낮음 (스트리밍) |

### 실제 측정 데이터 (3개 워커 스레드)

## 🎯 사용 시나리오별 권장 버전

### BeatDetectorV0 - 빠른 테스트용
- ✅ **용도**: 비트 감지 알고리즘 검증, 성능 테스트
- ✅ **장점**: 가장 빠른 처리 속도 (~50초)
- ⚠️ **단점**: 낮은 정확도, 낮은 분해능 (50ms)
- 📊 **예상 성능**: ~150초 순차 → ~50초 멀티스레드 (3배)

### BeatDetectorV1 - 일반 음악용 (현재 선택)
- ✅ **용도**: 발라드~댄스곡, 대부분의 일반 음악
- ✅ **장점**: 중간 정확도, 중간 속도, 안정적
- ⚠️ **단점**: 높은 정확도 필요 시 부족할 수 있음
- 📊 **실제 성능**: **332초 순차 → 112.8초 멀티스레드 (2.94배)**
- 🔬 **특징**: 높은 병렬화 효율 (98.1%), 최소 오버헤드 (1.9%)

### BeatDetectorV2 - 정확도 중심
- ✅ **용도**: 복잡한 시간 서명, 높은 정확도 필요
- ✅ **장점**: 가장 높은 정확도, 스트리밍 처리로 메모리 효율
- ⚠️ **단점**: 느린 처리 속도
- 📊 **예상 성능**: ~280초 순차 → ~95초 멀티스레드 (약 3배)
