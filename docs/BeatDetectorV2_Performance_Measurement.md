## 🔄 BeatDetectorV2 알고리즘 특성

**처리 방식**:
- Streaming 청크 기반 처리 (FFT 링 버퍼)
- Dual ODF 아키텍처:
  - `odfTempo`: 자동 상관 + Log-Normal Prior로 BPM 추정
  - `odfTrack`: 동적 프로그래밍으로 위상 추적 및 다운비트 감지
- Hann Window 캐싱으로 최적화
- Kick 밴드 가중치 (1.5x) 적용
- 24밴드 로그 필터뱅크

**특징**:
- 가장 높은 정확도
- 복잡한 시간 서명 처리 가능
- 스트리밍 처리로 메모리 효율
- PCM 버퍼 크기 제한 (OOM 방지)
- 높은 계산 복잡도로 인한 느린 처리

## 🎯 최적화 기회

### 단기 (구현 가능)
1. **병렬화 개선**
   - 4개 워커 스레드로 확장
   - 예상 성능: ~205초 (약 33% 개선)

2. **FFT 최적화**
   - NEON SIMD 활용 (ARM)
   - Radix-4 FFT 알고리즘 검토

### 중기
1. **Hann Window 재사용**
   - 현재 캐싱 적용 ✓
   - 추가 재사용 기회 검토

2. **DP 탐색 공간 축소**
   - 비트 주파수 후보 범위 제한
   - 상태 전이 제약 강화

### 장기
1. **GPU 가속**
   - FFT 연산 GPU 오프로딩
   - NNAPI를 통한 신경망 가속

2. **적응형 hopMs**
   - 곡 특성에 따라 hopMs 동적 조정
   - BPM 범위에 따른 최적화
