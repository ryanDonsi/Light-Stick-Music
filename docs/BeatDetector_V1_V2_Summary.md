# BeatDetector V1 vs V2 - 비교 요약

**기준**: 33곡 멀티스레드 환경 (3개 워커)

---

## BeatDetector V1/V2 종합 비교표

| 항목 | **BeatDetectorV1** | **BeatDetectorV2** |
|------|------------------|------------------|
| **ODF (특징 추출)** | IIR 3밴드 에너지 | Dual ODF: SuperFlux(Tempo) + SuperFlux(Track) |
| **BPM 추정** | Autocorrelation + Log-Normal Prior | DBN HMM Forward (Viterbi-max) |
| **Hop 크기** | 10ms | 10ms |
| **하모닉 보정** | half-tempo ratio 체크 (60%) | 910ms guard + combPriorScore |
| **외부 의존성** | 없음 | JTransforms FFT 라이브러리 |
| **정확도 (가중평균)** | 81.0% | 85.2% |
| **메모리 사용** | ~30MB (3분 곡) | ~32KB (링버퍼 스트리밍) |
| **BPM 범위** | 모든 범위 (연속 스윕) | 120BPM 중심 (확률 모델) |
| **비표준 BPM** | 우수 (66, 140+ BPM) | 기본값 선호 |
| **처리 시간 (순차)** | 332초 | 712초 |
| **처리 시간 (멀티)** | **112.8초** | **273.5초** |
| **속도 향상** | 2.94배 | 2.60배 |
| **병렬화 효율** | 98.1% | 86.8% |
| **주요 강점** | 빠른 속도, 정확한 BPM | Dual ODF로 높은 정확도, 메모리 효율 |
| **주요 약점** | 낮은 정확도 | 느린 처리, 제한된 BPM 범위 |

## 권장 사항

| 상황 | 권장 | 이유 |
|------|------|------|
| **K-POP, 표준 BPM (80~160)** | ✅ **V1** | 빠르고 안정적, 1분 53초 |
| **클래식, 비표준 BPM (66 BPM)** | ✅ **V1** | 정확한 BPM 감지 |
| **정확도 중심** | V2 | 최고 정확도, 4분 34초 |
| **프로덕션 (현재 선택)** | ✅ **V1** | 가장 효율적 (98.1%), 실용적 |

## 현재 최적 구성

### ⭐ **BeatDetectorV1 + 3개 워커 스레드**

- 처리 시간: **112.8초** (33곡 / 104.5분 음악)
- 병렬화 효율: **98.1%** (거의 완벽)
- 오버헤드: **1.9%** (극히 낮음)
- V2 대비: **2.4배 빠름**
