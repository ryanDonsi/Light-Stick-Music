# Beat Accuracy Checker Tool 소개

**목적**: BeatDetector 알고리즘의 정확도를 객관적으로 측정하고 비교 분석하는 도구  
**버전**: 1.0  
**최종 업데이트**: 2026-06-24

---

## 🎯 개요

Beat Accuracy Checker는 음악 파일의 비트 감지 정확도를 여러 GT(Ground Truth) 엔진과 비교하여 측정하는 테스트 도구입니다. BeatDetectorV0, V1, V2의 성능을 객관적으로 평가하고 개선 효과를 정량화합니다.

---

## 🔧 주요 기능

### 1. 다중 GT 엔진 지원
- **madmom**: CNN+HMM 기반 정확도 높음 (F1: 85.6%)
- **Beat Transformer**: Transformer 기반 최고 정확도 (F1: 88.5%)
- **librosa**: 빠른 처리 속도 (F1: ~70%)

### 2. 포괄적 평가 지표
- **F-Measure**: 70ms 허용 오차 기준
- **Precision**: 거짓양성 비율 측정
- **Recall**: 거짓음성 비율 측정
- **BPM 오차율**: 추정 BPM 정확도
- **등급 분류**: S/A/B/C/D 등급 자동 판정

### 3. 대량 분석 처리
- 배치 처리 모드 (여러 곡 동시 분석)
- 멀티스레드 병렬 처리 지원
- 처리 시간 측정 및 병렬화 효율 분석

### 4. 상세 리포팅
- 곡별 정확도 상세 분석
- 장르별 통계 집계
- V1/V2 비교 테이블 자동 생성

## 📊 평가 기준

### 등급 기준 (F-Measure 기반)

| 등급 | 범위 | 평가 |
|------|------|------|
| **S** | 90% 이상 | 매우 우수 |
| **A** | 80~89% | 우수 |
| **B** | 70~79% | 양호 |
| **C** | 60~69% | 보통 |
| **D** | 60% 미만 | 미흡 |

### 정확도 가중치

```
종합 정확도 = madmom(60%) × F1 + Beat Transformer(40%) × F1
```

- madmom: 일반적 정확도 측정 (규칙적 비트에 강함)
- Beat Transformer: 고정확도 측정 (복잡한 패턴 분석)

## 💡 사용 방법

### 기본 사용법

```python
# 단일 곡 분석
accuracy_checker.analyze("song.mp3", detector_version="v1")

# 여러 곡 배치 분석
accuracy_checker.analyze_batch(
    audio_files=["song1.mp3", "song2.mp3"],
    detector_version="v2",
    gt_engines=["madmom", "beat_transformer"]

# 결과 생성
accuracy_checker.generate_report(format="csv", output_dir="./results")
```

## 🔍 정확도 해석

### F-Measure 의미

**F-Measure = 2 × (Precision × Recall) / (Precision + Recall)**

- **높은 Precision**: 감지된 비트의 대부분이 정확함
- **높은 Recall**: 실제 비트의 대부분을 감지함
- **높은 F-Measure**: 정확하면서도 빠짐없는 감지

### BPM 오차율

```
BPM 오차 = |추정 BPM - 실제 BPM| / 실제 BPM × 100%

예: 실제 120 BPM → 추정 130 BPM = 8.3% 오차
```
