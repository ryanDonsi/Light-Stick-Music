## ⚙️ 5. 알고리즘 개선 분석

### 주요 기술 개선

| 항목 | V0(V8) | V1 | 개선 내용 |
|------|---------|----|---------.|
| **ODF** | IIR 3밴드 | IIR 3밴드 | 동일 (복잡도 유지) |
| **Hop 크기** | 50ms | 10ms | **5배 향상** (분해능) |
| **BPM 추정** | Autocorrelation | Autocorrelation + Log-Normal Prior | **Prior 추가** (안정성) |
| **입력 방식** | Envelope (사전계산) | PCM (직접입력) | **직접입력** (유연성) |
| **특징** | 저분해능 | 고분해능 + Prior 기반 | **기초부터 재설계** |

### 개선된 알고리즘 흐름

```
V0: Envelope → 50ms hop → AC → 단순 BPM 선택
                          └→ 비트 추적 (오류율 60%)

V1: PCM → IIR 필터 → 10ms hop → AC + Log-Normal Prior
                               └→ 반박자 체크 (60%)
                               └→ 비트 추적 (오류율 9%)
```

## 💡 7. 성공 요인 분석

### V1 개선의 핵심

1. **Hop 크기 최적화** (50ms → 10ms)
   - 5배 높은 시간 분해능
   - 세밀한 비트 위치 감지 가능

2. **Log-Normal Prior 추가**
   - 120 BPM 중심 사전 지식
   - BPM 범위 제약으로 오류 감소
   - 확률 기반 추정으로 안정성 향상

3. **반박자 체크 추가**
   - 140+ BPM 곡에서 half-tempo 오류 방지
   - `ratio ≥ 0.60` 기준으로 올바른 템포 선택

4. **직접 PCM 입력**
   - Envelope 사전계산 제거
   - 원본 신호의 정보 손실 방지
   - 유연한 처리 가능

### 개선 효과 검증

| 개선 사항 | 기여도 추정 |
|----------|-----------|
| Log-Normal Prior | **40%** (BPM 안정화) |
| Hop 크기 (50→10ms) | **30%** (분해능 향상) |
| 반박자 체크 | **20%** (특수 곡 대응) |
| 직접 PCM 입력 | **10%** (신호 품질) |
